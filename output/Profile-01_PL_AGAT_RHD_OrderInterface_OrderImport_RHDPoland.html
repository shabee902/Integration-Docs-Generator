    Event-based HTTP POST to `AGAT/Order` with `application/xml` payload.
  2. Parse XML
    Read and validate structure.
  3. Validate Data
    Check required fields and formats.
  4. Map Fields
    Apply mapping rules (89 pairs).
  5. Forward / Handoff
    On success, hand off to `02_PL_AGAT_RHD_OrderInterface_StatusResponse_RHDPoland`.
  6. Error Handling
    On error, route to `02_PL_AGAT_RHD_OrderInterface_StatusResponse_RHDPoland`.

Profile Purpose

Overview
This profile receives POST requests at AGAT/Order.

Input Data

1. Where the data comes from
Requests are sent to the endpoint `AGAT/Order`.
2. How the data is sent
The system expects an HTTP POST request with the format application/xml.
3. What must be included
Each request contains a single order message that follows the agreed XML structure and includes the required fields (for example: order number, customer, items, quantities, and dates).
